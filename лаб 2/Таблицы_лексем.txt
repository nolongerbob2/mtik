Таблицы лексем для подмножества C++ по конструкциям из test.cpp (лабораторная 1).
Используются лексическим анализатором (lexer.cpp).

-------------------------------------------------------------------------------
Таблица 1 — Ключевые слова (KEYWORD)
-------------------------------------------------------------------------------
id | Лексема  | Комментарий
 1 | include  | Директива препроцессора #include
 2 | static   | Класс хранения / статический член
 3 | const    | Неизменяемая сущность
 4 | int      | Целочисленный тип
 5 | bool     | Логический тип
 6 | void     | Отсутствие значения (тип функции)
 7 | double   | Вещественный тип
 8 | if       | Условный оператор
 9 | else     | Ветвь else
10 | for      | Цикл for
11 | while    | Цикл while
12 | return   | Возврат из функции
13 | break    | Выход из цикла / switch

-------------------------------------------------------------------------------
Таблица 2 — Идентификаторы (IDENTIFIER)
-------------------------------------------------------------------------------
Правило: последовательность латинских букв, цифр и «_», начинается с буквы или «_».
Примеры из test.cpp: kLimit, add, main, in_range, print_sum, cout, endl, std,
vector, n, sum, prod, ratio, flag, x, lo, hi, s, i, v, acc, static_cast (как имя).

-------------------------------------------------------------------------------
Таблица 3 — Целочисленные константы (CONSTANT_INT)
-------------------------------------------------------------------------------
Правило: последовательность десятичных цифр.
Примеры: 42, 5, 0, 1, 2, 3, 100

-------------------------------------------------------------------------------
Таблица 4 — Вещественные константы (CONSTANT_FLOAT)
-------------------------------------------------------------------------------
Правило: целая часть и/или дробная часть, разделённые одной точкой (подмножество C++).
Примеры из test.cpp: 0.0

-------------------------------------------------------------------------------
Таблица 5 — Строковые константы (CONSTANT_STRING)
-------------------------------------------------------------------------------
Правило: символы в двойных кавычках; допускаются escape-последовательности \", \\, \n, \t, \r.
Пример: "sum="

-------------------------------------------------------------------------------
Таблица 6 — Булевы константы (CONSTANT_BOOL)
-------------------------------------------------------------------------------
id | Лексема | Комментарий
 1 | true    | Истина
 2 | false   | Ложь

-------------------------------------------------------------------------------
Таблица 7 — Операторы (OPERATOR)
-------------------------------------------------------------------------------
Двухсимвольные и более (примеры из test.cpp):
::   область видимости
<<   вывод в поток
>>   ввод из потока (в тесте не используется, поддерживается)
++   инкремент
--   декремент
==   равенство
!=   неравенство
<=   меньше или равно
>=   больше или равно
&&   логическое И
||   логическое ИЛИ
+=   сложение с присваиванием
-=   вычитание с присваиванием
*=   умножение с присваиванием
/=   деление с присваиванием
%=   остаток с присваиванием
Односимвольные: = + - * / % < > ! ?

-------------------------------------------------------------------------------
Таблица 8 — Разделители (DELIMITER)
-------------------------------------------------------------------------------
( ) { } [ ] ; , . : #
Символы < и > в реализации лексера отнесены к операторам (сравнение, шаблоны vector<int>,
#include <iostream>), чтобы корректно различать < и <<, > и >>.

-------------------------------------------------------------------------------
Правила разделения лексем (кратко)
-------------------------------------------------------------------------------
1. Пробельные символы (пробел, таб, перевод строки) разделяют лексемы и не входят в токены.
2. Идентификаторы и ключевые слова заканчиваются на первом символе, не являющемся буквой, цифрой или «_».
3. Литералы чисел считываются по правилам целого/вещественного; после числа буква — ошибка; две точки подряд — ошибка.
4. Строка в "..." — одна лексема до закрывающей кавычки (с учётом escape).
5. Операторы распознаются по максимальной длине (например, << раньше, чем <).
6. Символы ( ) { } ; , # и др. в таблице разделителей образуют отдельные токены-разделители.
7. Символы < > распознаются как операторы (одиночные или в составе <<, >>, <=, >=).
